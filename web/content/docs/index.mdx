Selection Room is a transparent College Football Playoff selection simulator. It builds a composite ranking from public inputs, selects a 12-team field under the published 5+7 rules, seeds a bracket, and shows its work at every step. The live site is at [www.selectionroom.org](https://www.selectionroom.org).

## Two ways to use it

**Browse the hosted site (no install).** Everything on [selectionroom.org](https://www.selectionroom.org) is open: pick a run from the header switcher, then explore the [Dashboard](/dashboard), [Rankings](/rankings), [Bubble](/bubble), [Bracket](/bracket), [Validation](/validation), and [Scenario Lab](/scenario-lab). Signing in with GitHub lets you launch your own live runs.

**Run it locally or contribute.** The whole project is open source: a Python engine plus this Next.js app. Start at [Getting started](/docs/getting-started) and [Local development](/docs/local-development).

## Product vs docs

| Surface | Purpose |
|---------|---------|
| [Dashboard](/dashboard) | Active run overview: projected field, matchups, bubble snapshot |
| [Rankings](/rankings) | Full composite table with score bars and the team resume drawer |
| [Bubble](/bubble) | Cut line, last four in / first four out, Selection Stability board |
| [Bracket](/bracket) | Seeded bracket views with image download |
| [Scenario Lab](/scenario-lab) | Reweight the composite and diff the projected field |
| [Validation](/validation) | Historical backtests plus the Model vs Committee comparison |
| [Methodology](/methodology) | In-app metric reference and live weights |
| Docs (here) | How the product works, what the model claims, and developer contracts |
